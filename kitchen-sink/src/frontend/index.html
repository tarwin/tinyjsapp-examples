      Tray & notifications menu-bar recipe · 0.6.0 rich notifications

      tiny.notify(title, body, { id, subtitle, sound }) — a packaged, signed .app shows a real Notification Center banner; clicking it routes back via tiny.app.onNotificationClick (dev falls back to osascript)

      Tray mode puts ◉ Deck in the menu bar with a live menu. The full recipe: tray.set() + win.setHideOnClose(true) + app.setDockVisible(false) — closing the window then hides it, and the app lives in the menu bar.

      Auto-update & packaging update.check / build --dmg / notarize

      Point tinyjs.json at a hosted manifest.json (made by tinyjs publish) and the app updates itself: sha256 + signature verified, .app swapped in place, relaunched. Ship it with tinyjs build --dmg (0.3.1: a drag-to-Applications disk image) and tinyjs notarize (notarytool submit + staple).

      this project has no update.url configured — the check will say so

      Window state & menu read-back 0.5.0 · getState / restore / setFullscreen / menu.get

      tiny.win.getState() reports position, size, every flag (fullscreen / minimized / visible / focused / on-top / resizable) and the screen it sits on — read live from the NSWindow
      the Actions ▸ Toggles submenu mirrors real app state via tiny.menu.update — checkmarks track these very buttons; read any item back with tiny.menu.get

      Deep links & file associations 0.4.0 · packaged .app only

      This build claims the tinydeck:// URL scheme and the .txt / .md / .log extensions in tinyjs.json. In a packaged app, open tinydeck://…, “Open With → Tiny Deck”, and dropping those files on the Dock icon all deliver events here — even on cold start (they’re buffered until the page is ready). A second launch just activates this window (single-instance). Dev mode has no bundle, so this only fires from the built .app; the backend can also handle it via export onOpenUrl / onOpenFiles.

      after `tinyjs build`, from a terminal:
open dist/Tiny\ Deck.app
open "tinydeck://compose?to=you&subject=hi"
open -a "Tiny Deck" ~/Desktop/notes.txt
      tiny.app.onOpenUrl / onOpenFiles events land here, live

      Window chrome 0.7.0 · frameless / transparent / vibrancy

      Reshape the native window live with tiny.win.setChrome({ frame, trafficLights, transparent, vibrancy }) — native resize, focus and fullscreen keep working. Go frameless and the deck’s top header becomes the titlebar (it carries data-tiny-drag): drag it to move the window, double-click it to zoom; buttons and inputs inside a drag region are excluded automatically. Packaged apps can bake the same shape into tinyjs.json ("chrome": {…}) so it applies before first paint — no titlebar flash. (transparent shows through only where the page background has alpha.)

      drag the top header · double-click it to zoom
      tiny.win.setChrome(…) restyles the frame while the window keeps every native behavior — getState() reports the chrome back too (0.7.0)
